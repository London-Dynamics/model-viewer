LD Modular by London Dynamics. This page showcases the LD Modular API through practical examples for placing GLBs into a scene, selecting and editing modular entities, and managing scene content throughout an interactive workflow.

Place a GLB into the scene

placeGLB() – place and register a GLB in the modular scene.

Interactively place a model, select it, and use the control strip while transform events stream in the debug readout. The code demonstrates how to configure a modular-ready <model-viewer>, call placeGLB(), and wire UI actions to placement and editing behavior.

Multi-select

selectAll() / deselectAll() / applyRectangleSelection() with Shift-click toggling.

Click objects to select one at a time, hold Shift while clicking to add or remove items from the selection, then drag or rotate the selection as a group. Seeded models use the Khronos Duck (offset origin) so rotation-disc behaviour matches the Rotation Controls example. Use Select all and Deselect all to exercise the programmatic APIs, or toggle the fixed selection rectangle and click Apply selection to call applyRectangleSelection() with its client coordinates. Switch between group and part scope to change what a click targets.

Room Surface Snap (Walls)

Constrain dragging with surfaceSnap so parts snap to valid wall surfaces.

Interactively drag window and door parts around the room and observe that placement is accepted only on supported walls. The code demonstrates room-aware model configuration, per-object snap options, and event wiring for constrained surface placement.

Transform objects and scene

Call rotate(), translate(), and scale() with an object's name to manipulate its transform. The name "scene" can be used to manipulate the entire model.

Use the sliders to rotate selected nodes and the full scene in real time. The code demonstrates mapping UI inputs to modular transform APIs and targeting either specific object names or the root scene node.

Rotation modes and snapping

Call rotate(), with an object's name to manipulate its rotation. The name "scene" can be used to manipulate the entire model. The rotation mode can be set to "relative" or "snapToClosest".

Click preset angle buttons to compare exact, relative, and snap-to-closest rotations on the same object. The code demonstrates passing rotation mode options into rotate() and building predictable snapping controls for editor-style workflows.

Load model source from buffer

setSrcFromBuffer() – replace the active model from an ArrayBuffer instead of a URL.

Interactively swap between preloaded model buffers using the control buttons and watch the viewer update without changing the source URL. The code demonstrates fetching serialized buffer payloads, converting them to ArrayBuffer, and calling setSrcFromBuffer().

Bulk Placement

placeManyGlb() – place multiple GLBs at once using known transforms.

Run bulk placement to instantiate a full layout and track progress in the panel while optional tree logging verifies the result. The code demonstrates preparing batched placement payloads and executing placeManyGlb() for deterministic multi-object scene setup.

Hover anchor metadata

Listen for hover-change and render metadata.id (fallback metadata.part.id) at the reported anchor center while hovering placed objects.
