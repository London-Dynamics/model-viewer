LD Measure by London Dynamics. Give the user the ability to measure the size of objects in the scene.

Display measurements

How measurement works: By default, the measurement overlay shows the total bounding box of the entire scene. To enable interactive measurement of individual objects, use the measure-objects attribute.

measure-objects can be set to * to allow clicking any selectable object, or to a comma-separated list of object names (e.g. name1,name2) to restrict measurement to those objects. If left empty, only the scene bounding box is measured.

Selection scope: The selection-scope attribute controls what can be selected for measurement:

  • scene: Only the whole scene can be measured.
  • group: Groups or the whole scene can be measured.
  • part: Individual parts, groups, or the scene can be measured.

The selection logic now supports both placed objects (from modular scenes) and regular model nodes.

Tip: If your model uses modular/placed objects (e.g. with the Puzzler feature), measurement and selection will work for those objects. Otherwise, it will work for any named node in the model.

Customise labels

Add slots to customise the labels for the measurements. The slots are named "ruler-width", "ruler-height", and "ruler-depth".

Grid shape lines

Add custom line shapes on top of the LD Measure grid using the new grid-shape API.

Room measure and grid

With src-is-room, use walls and floor (Lit booleans, default false) to show tagged wall_* nodes, a walls group, a floor node, or floor_* nodes. This example starts with both on. Enable debug to log effective flags on load and when toggling. Use the Measurements switch to hide dimension lines and measure UI while keeping the grid. Measurement and grid are independent of room shell visibility.
