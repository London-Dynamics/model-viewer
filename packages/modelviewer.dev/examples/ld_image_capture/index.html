LD Image Capture by London Dynamics. Capture the current view as a data URL with optional width, height, file type, camera pose, and crop using captureImage().

Capture Image

Capture the current view as PNG or JPEG. The data URL is shown in the console and can be used as an image src.

Capture with camera pose

Pass getCameraJSON().object into captureImage({ camera }) (include controlsSnapshot if the saved view was panned). The pose is applied for one render, then the on-screen camera is restored.

Capture with crop

Use crop: { x, y, width, height } (in source canvas pixels) to capture only a region. Optionally combine with width and height to scale the crop.

Background color

Use backgroundColor so formats like JPEG (no transparency) show a solid background instead of black. The offscreen buttons exercise the width×height render-target path (the bug case); cornflowerblue must appear, not black / transparent.

fitToBox without camera (stay put)

Orbit to a custom framing, then capture with { width, height, fitToBox: true } and no camera. The thumbnail may reframe the model for the capture; the live viewer must stay where you left it. Status reports whether the pose was restored.

Orthographic views (above, front, side)

Use setCameraType('orthographic') and rotateCamera(azimuth, polar, false) to set each view, then getCameraJSON().object for each pose. Pass those objects into captureImage({ camera }) to capture without changing the visible view. Click “Prepare views” once, then capture from above, front, or side.
